# LILLIPUT Decoder (Stim + PyMatching + Vivado)

This repo contains Jupyter notebooks to prototype and evaluate a **LILLIPUT-style LUT decoder** workflow using **Stim** and **PyMatching**, and to generate LUT/test-vector files for **Vivado/FPGA simulation**.

---

## Repo Structure

- `notebooks/` — all experiments + LUT generation
- `verilog/` — hardware decoder + testbench (`lilliput_decoder.sv`, `tb_lilliput.sv`)
- `generated_files/` — LUT + vectors generated by `gen_lut.ipynb`
  - `lut_synd.mem`, `lut_out.mem`, `tb_vectors.mem`

---

## Contents (Notebooks)

### 1) `LILLIPUT-without-sliding-window.ipynb` (Base / starting point)
- **Start here** (baseline pipeline).
- Gets about **0–2% logical error rate** for **~1% phenomenological error** (relative to the sliding-window attempt).

### 2) `LILLIPUT-with-sliding-window.ipynb` (Unoptimized / failed trial)
- Unoptimized trial/failed approach toward paper-style sliding-window decoding.
- Gives about **4–8% logical error** for **~1% phenomenological error**.
- Included for reference/future improvement.

### 3) `LILLIPUT_round_weight_table.ipynb` (Comparison table generator)
- Generates a comparison table:
  - **Rows = QEC rounds**
  - **Columns = max_weights**
- Requires `reportlab`.

### 4) `gen_lut.ipynb` (Vivado LUT + test vectors generator)
Generates files used by the SystemVerilog decoder/testbench:
- **`generated_files/lut_synd.mem`** — LUT syndromes (hex, packed detector bits)
- **`generated_files/lut_out.mem`** — LUT outputs (0/1 predicted logical observable bit)
- **`generated_files/tb_vectors.mem`** — testbench vectors (`<syndrome_hex> <obs_bit>`), default **1000** samples
